---
name: using-playbooks
description: Use when starting work or when a request changes shape and you need to load the minimal relevant skill set before acting
---

If a skill clearly applies, or the user names one, read it before substantive work.

Rules:
- Load the minimum set that matches the task. Do not add extra process once scope is already clear.
- Check for matching skills before exploring files, planning, or asking clarifying questions.
- Prefer the skill that decides approach first: debugging or brainstorming before execution; execution before review or finish.
- Do not bounce back into brainstorming or planning when the user already supplied an approved plan or a bounded patch.
- If a skill has a checklist, track it with the available plan or todo tool.
- If you think you remember a skill, read the current file anyway.

User instructions define the goal. Skills define how to work.
